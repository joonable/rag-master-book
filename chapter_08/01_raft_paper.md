# [Chapter 8.1] RAFT 논문 심층 분석: RAG 전용 파인튜닝

RAFT(**R**etrieval-**A**ugmented **F**ine-**T**uning)는 모델이 단순히 지식을 외우는 것을 넘어, **"제공된 문서(Context)를 얼마나 잘 활용하는가"**에 초점을 맞춘 파인튜닝 기법입니다.

## 1. RAFT가 해결하려는 문제 (Motivation)
일반적인 LLM 파인튜닝(SFT)과 RAG 사이에는 괴리가 존재합니다.
- **표준 SFT:** 모델의 내부 지식(Parametric Memory)을 늘리지만, 외부 문서를 참고하는 법을 배우지 못합니다.
- **표준 RAG:** 외부 문서를 주지만, 모델이 문서 내의 노이즈(관련 없는 정보)에 휘둘리거나 문서 내용을 무시하고 아는 대로 대답하는 경향이 있습니다.

RAFT는 이 두 방식의 장점을 결합하여 **"문서에 근거하여 사고하는 모델"**을 만듭니다.

## 2. 핵심 훈련 데이터 구성 (Data Composition)
RAFT는 학습 시 각 질문($Q$)에 대해 $n$개의 문서를 제공합니다. 이때 문서는 두 가지 유형으로 나뉩니다.

### A. Oracle 문서 ($P$): 정답을 포함한 진짜 문서
### B. Distractor 문서 ($D$): 정답과 상관없는 가짜(방해) 문서

학습 데이터셋은 다음 세 가지 비율로 섞어서 구성합니다:
1. **유형 1 (Oracle + Distractors):** 질문에 대한 정답 문서 1개와 방해 문서 $n-1$개를 함께 제공합니다. (대부분의 케이스)
   - *목표:* 수많은 정보 중 정답을 정확히 추출(Extraction)하는 능력을 기릅니다.
2. **유형 2 (Distractors Only):** 정답 문서 없이 오직 방해 문서만 제공합니다.
   - *목표:* 문서에 답이 없을 때 아는 척하지 않고 "답을 찾을 수 없음"을 배우거나, 자신의 내부 지식 대신 문서에 의존하는 법을 배웁니다.
3. **유형 3 (Standard SFT):** 문서 없이 질문만 제공합니다.

## 3. 생각의 사슬 (Chain of Thought, CoT) 설계
RAFT의 가장 큰 특징은 답변의 **형식**입니다. 모델은 정답을 말하기 전에 반드시 문서에서 증거를 먼저 찾아야 합니다.

### 답변의 구조 예시:
> **질문:** 2024년 삼성전자의 AI 전략 핵심은 무엇인가요?
> 
> **모델의 답변 프로세스 (RAFT 스타일):**
> 1. **[증거 추출]:** 문서 2에 따르면 "삼성전자는 '모두를 위한 AI' 비전 아래 온디바이스 AI를 전 가전으로 확대한다"라고 명시되어 있음.
> 2. **[논리적 추론]:** 이는 하드웨어 경쟁력을 바탕으로 클라우드 연결 없이도 작동하는 독자적인 생태계를 구축하려는 의도로 풀이됨.
> 3. **[최종 답변]:** 삼성전자의 2024년 핵심 전략은 온디바이스 AI의 전방위적 확대와 이를 통한 독자 생태계 강화입니다.

이러한 **[추출] -> [추론] -> [답변]** 단계를 학습함으로써 모델은 할루시네이션을 획기적으로 줄입니다.

## 4. RAFT의 핵심 알고리즘 요약
- **Retrieval-Aware:** 모델이 '검색된 결과가 항상 완벽하지 않다'는 것을 인지한 상태에서 훈련됩니다.
- **Verbatim Citations:** 문서의 문구를 그대로 인용하는 훈련을 통해 근거 기반 답변 능력을 강화합니다.
- **Domain Specificity:** 특정 도메인(법률, 의료, 사내 문서 등)의 문서 구조와 전문 용어에 최적화된 추론 능력을 갖추게 됩니다.

## 5. 학습 효과 비교
| 비교 항목 | 일반 LLM | 일반 RAG | RAFT 적용 모델 |
| :--- | :---: | :---: | :---: |
| 내부 지식 활용 | 높음 | 보통 | 보통 |
| 외부 문서 참조 | 낮음 | 높음 | **매우 높음** |
| 노이즈 저항성 | 낮음 | 보통 | **매우 높음** |
| 할루시네이션 | 높음 | 보통 | **매우 낮음** |
